# CryptoPets:鏈上寵物養成DApp

一個結合 **ERC-721(NFT)** 與 **ERC-1155(SFT)** 技術的混合式Web3養成戰鬥遊戲。玩家可以透過遠征收集素材、進行即時戰鬥，並在區塊鏈上安全地交易資產。

---

## 專案簡介

本專案旨在打造一個輕量化且具備完整經濟循環的寵物養成系統。透過「鏈下執行遊戲邏輯、鏈上確認資產權屬」的混合模式，兼顧遊戲流暢度與區塊鏈的去中心化特性。

- **五大元素寵物：** 包含五種屬性的寵物，每種具備三階段進化型態。
- **素材系統：** 分為E、D、C、B、A五個等級，用於寵物進化，且還有經驗素材分為C、B、A。

## 技術棧(TechStack)

- **智能合約:** Solidity(OpenZeppelin標準庫)
- **前端開發:** HTML5,CSS3,JavaScript(ES6+)
- **遊戲引擎:** Phaser3(用於戰鬥畫面與特效渲染)
- **Web3交互:** Ethers.js/Web3.js
- **錢包集成:** MetaMask
- **存儲機制:**
  - **鏈上:** 寵物數據、素材餘額、交易紀錄、進化階段
  - **鏈下:** LocalStorage(用於冷卻計時與暫存狀態)

## 核心物件資料結構(Data Schema)

本專案採用混合式架構，資產權屬由鏈上定義，遊戲邏輯與數值由鏈下JS運算。

### 1.寵物NFT

記錄寵物的個體差異與進化狀態，支援動態`tokenURI`變換。

```javascript
{
    "id": "",           // 唯一的 Token ID
    "name": "",        // 寵物名稱
    "element": 1,           // 屬性索引 (1:火, 2:水, 3:草, 4:光, 5:暗)
    "stage": 0,             // 進化階段 (0:初階, 1:中階, 2:高階)
    "TokenURI":"",
    "stats": {
        "iv": 95,             // 個體值 (0-100) 出生時隨機
        "hp": 120,            // 生命值
        "atk": 45,            // 攻擊力
        "def": 30             // 防禦力
    },
    "exp": 1500,            // 累計經驗值 TODO
    "owner": "",    // 持有者錢包地址
    "birthTime": "" // 出生日期
}
```

### 2.素材SFT

採用批次代幣管理，ID 規則採「屬性+等級」前綴編碼（例如：MAT-1C 為火系 C 級）。

```Javascript
{
    "id": "MAT-1C",         // 素材ID (MAT-屬性等級)
    "name": "",      // 素材名稱
    "element": 1,           // 對應屬性 (1:火, 2:水, 3:草, 4:光, 5:暗)
    "grade": "C",           // 等級 (E, D, C, B, A)
    "amount": 0,           // 玩家當前持有的數量
    "description": ""
}
```

### 3. 回憶錄

紀錄寵物的生平事蹟，即使用戶已將其賣出，仍可透過ID追蹤其榮譽。

```Javascript
{
    "tokenId": "",      // 對應之寵物 ID
    "totalBattles": 0,     // 總戰鬥次數
    "winRate": 0.65,        // 勝率 (0.65 = 65%)
    "adventures": 0,       // 遠征次數
    "prevOwners": "",     // 賣給的最後一個人
    "status": "Active"      // 狀態: 持有中 (Active) / 已賣出 (Sold)
}
```

### 進化邏輯設計

系統透過 EVOLUTION_CONFIG 矩陣進行進化判定，支援多重素材融合與數值加成。

```JavaScript
const EVOLUTION_CONFIG = {
    "ELEMENT_1": {
        "TO_STAGE_1": { "requiredMat": [{"minLevel":"","id":"" , "count":""}], "statsBoost": 0 },
    },
};
```

## 遠征與素材掉落 (Endless Adventure System)

本系統採「無止盡遠征」模式，寵物一旦出發將持續累積獎勵，直到玩家手動執行「領取並上鏈」動作。

### 1. 遠征運行機制

- **掛機收益：** 寵物出發後，每小時根據關卡效率產生基礎獎勵。
- **獎勵暫存：** 獎勵優先儲存於LocalStorage，此時不消耗任何 Gas。
- **上鏈領取 (Claim)：** 玩家點擊領取時，系統計算累積時數，並一次性將所有素材 SFT 與經驗值寫入區塊鏈。

## 系統架構(SystemArchitecture)

### 1.鏈上資產層(On-Chain)

- **寵物NFT(ERC-721):** 記錄寵物的唯一數據（等級、屬性、進化階段），支援動態更新`tokenURI`變更外觀。
- **素材SFT(ERC-1155):** 管理五種等級的升級素材，利用批次傳輸特性降低交易GasFee。
- **交易合約(TradingContract):** 實現原子化交易(AtomicSwaps)，確保P2P交易過程的安全與透明。

### 2.鏈下邏輯層(Off-Chain)

- **戰鬥引擎:** 使用JavaScript編寫傷害運算公式與屬性相剋邏輯，由Phaser3渲染2.5D戰鬥畫面。
- **遠征系統:** 利用`localStorage`紀錄時間戳記，結合前端校時算法得出掉落結果。
- **進化控制器:** 當素材充足時，導引玩家簽章呼叫合約進行`burn`(銷毀素材)與`upgrade`(升級NFT)。

## 遊戲畫面與功能

### 登入介面(Login/Auth)

- **自動偵測:** 系統會自動辨別瀏覽器是否安裝MetaMask。
- **網路防錯:** 若使用者未連接至指定網路（如Sepolia測試網），將跳出引導提示進行切換。

### 背包系統(Inventory)

- **素材管理:** 顯示透過遠征或戰鬥獲得的素材，每種素材堆疊上限為64個。
- **資產同步:** 提供「上鏈同步」功能，讓玩家批次將鏈下暫存素材轉為鏈上SFT代幣。

### 市場畫面(Marketplace)

- **自由定價:** 玩家可自由選擇想賣出的素材種類與數量，並自訂售價。
- **授權機制:** 整合`setApprovalForAll`授權，確保成交瞬間資產能安全且自動地完成轉移。
